## 4. 内容与支持分工

| 问题类型 | 用什么解决 | 原因 |
| --- | --- | --- |
| 稳定重复知识 | 录播 / 文档 / 模板 | 可自助复用 |
| 需要判断 | 诊断 / Office Hour | 无法只靠内容 |
| 操作失败 | 检查清单 / 错误库 | 快速定位 |
| 个性化问题 | 有边界的一对一 | 控制支持成本 |
| 同类共性问题 | 公开答疑 / FAQ | 一次解决多人 |

## 8. 反馈与证据

### 基线

- 用户现在如何完成任务？
- 花多长时间？
- 当前结果是什么？
- 最大阻塞是什么？

### 过程

- 完成了哪些动作？
- 哪一步失败？
- 使用了哪些材料和支持？

### 结果

- 最终提交物；
- 前后对比；
- 用户行为变化；
- 可以公开的截图或链接；
- 下一步。

## 9. 授权与隐私

- [ ] 联系方式不进入公开案例
- [ ] 公司 / 客户数据已脱敏
- [ ] 用户明确选择公开、匿名、内部或不授权
- [ ] 退出服务不影响授权选择
- [ ] 作品版权和使用范围已说明

## 10. 交付后产品化

| 重复内容 | 处理方式 |
| --- | --- |
| 重复讲解 | 录播 / 文档 |
| 重复判断 | 决策树 / Skill |
| 重复报错 | 错误库 / 检查清单 |
| 重复任务 | 模板 / 自动化 |
| 重复答疑 | FAQ / 固定 Office Hour |

只有重复出现的内容才进入标准产品。
