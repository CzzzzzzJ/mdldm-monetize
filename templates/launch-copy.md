# Launch Copy Pack

## 使用原则

- 文案必须基于真实问题、真实结果和真实交付；
- 不使用虚假倒计时、虚假名额、伪造购买记录；
- 首发的任务是验证，不是制造“卖爆了”的形象；
- 不把用户的焦虑放大成恐吓；
- 价格、人数、周期和边界必须与实际一致。

## 1. 公开招募长文

### 标题

```text
[目标用户]，如果你正在 [具体场景]，我想找 [人数] 人一起完成 [结果]
```

### 正文

```markdown
最近我反复看到一个问题：

[用用户原话描述问题，不先介绍自己]

我自己曾经通过 [能力 / 案例] 解决过这个问题，也和 [访谈人数] 位用户聊过。大家真正卡住的不是 [表面问题]，而是 [核心阻塞]。

所以我准备做一期 [产品形态]：

- 适合谁：
- 在多长时间内：
- 一起完成什么结果：
- 如何验收：
- 我会提供什么：
- 你需要投入什么：
- 首期人数：
- 验证价格：

这期不适合：

- ...
- ...

它不是 [明确不包含内容]，也不承诺 [不可控结果]。

如果你正在真实遇到这个问题，可以 [平台允许的行动方式]。我会先确认是否匹配，再决定是否加入。
```

## 2. 短版招募

```text
我在找 [人数] 位 [目标用户]，用 [周期] 一起完成 [可验收结果]。

适合：正在 [具体场景]，已经尝试过 [当前替代方案]，但卡在 [核心问题] 的人。

交付：[3项以内]
验证价格：[价格]
不适合：[关键排除条件]

如果你确实正在解决这个问题，可以 [行动方式]。
```

## 3. 私聊邀请

```text
你好，我记得你之前提到过 [具体问题]。

我正在测试一期 [产品形态]，目标是在 [周期] 帮 [目标用户] 完成 [可验收结果]。因为你确实遇到过这个问题，所以想先问问你现在是否还在解决它。

如果已经解决，不用回复；如果还在，我可以先把完整范围和价格发你，你再判断是否合适。
```

禁止群发给没有表达过相关问题的人。

## 4. 访谈邀请

```text
我正在研究 [目标用户] 在 [场景] 遇到的 [问题]，想了解大家现在是怎么解决的。

不是推销，访谈大约 [时长]。我主要想问最近一次问题、你试过什么，以及什么结果才算真的解决。

如果你最近确实遇到过，愿意聊一下吗？
```

## 5. 拒绝后的跟进

```text
收到，谢谢你直接告诉我。

为了避免我误判，方便的话只想确认一个原因：是这个问题现在不急、结果不够清楚、交付方式不合适，还是价格和价值不匹配？不方便回复也没关系。
```

不要连续追问或施压。

## 6. 开营 / 启动通知

```markdown
# 欢迎加入 [产品名]

这期只有一个共同目标：[可验收结果]。

## 你需要完成

- 基线记录：
- 核心任务：
- 最终提交：

## 我会提供

- ...
- ...

## 节奏

- 启动：
- 任务节点：
- 答疑：
- 验收：

## 提问规则

- 入口：
- 响应时间：
- 不包含：

第一步请在 [时间] 前完成：[唯一动作]。
```

## 7. 结营成果邀请

```text
这次你从 [基线] 到 [结果] 的变化很有代表性。

我想把这段过程整理成案例，供后续用户理解交付方式。你可以选择：

1. 允许公开并展示昵称；
2. 允许公开但匿名；
3. 只用于内部复盘；
4. 不授权使用。

无论选择哪一种，都不会影响你已经获得的服务。
```

## 发布前自检

- [ ] 没有虚假稀缺和收入承诺
- [ ] 人群、问题和结果具体
- [ ] 有真实价格与真实容量
- [ ] 写清不适合谁
- [ ] 写清用户需要投入什么
- [ ] CTA 符合当前发布平台规则
- [ ] 公开案例已获得授权
