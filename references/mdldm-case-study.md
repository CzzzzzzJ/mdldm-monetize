# MDLDM Case Study

## 用途

本案例用于解释这套方法如何从真实业务中形成。它不是要求其他创作者复制麦当的产品组合，也不是收入承诺。

## 背景

麦当 mdldm 的路径不是先设计一个“知识付费商业模式”，而是从持续分享 AI 工具、工作流和项目实践开始，逐步形成课程、会员、群、答疑、工具权益和知识站。

## 第一阶段：内容与信任

早期价值来自：

- 快速筛选 AI 工具和变化；
- 用普通人能理解的方式讲清操作；
- 公开自己真实使用和项目过程；
- 帮用户降低信息筛选和试错成本。

这一阶段最重要的资产不是课程数量，而是“麦当能帮我判断该学什么、该怎么动手”的信任。

## 第二阶段：课程与服务组合

知识站课程成为主要入口，但用户实际使用的价值逐渐扩展到：

- 课程；
- VIP 群；
- 答疑；
- 更新通知；
- 项目案例；
- AI 工具和麦子额度；
- 飞书知识库等配套服务。

这说明用户购买的不是单门课程，而是一套“方向、行动、工具和陪跑”系统。

## 第三阶段：用户反馈暴露结构问题

2026 年 7 月的一轮匿名聚合反馈中：

- 收到 29 份有效反馈；
- 平均推荐分为 4.69 / 5；
- 用户高度认可小白友好、方向判断、跟上变化和工具试错成本；
- 用户最集中提出的问题是课程多、路径不够清楚、看完不知道下一步；
- 强需求集中在 Codex / Claude Code 深度实战、商业化闭环、AI 产品和真实项目。

高评分并没有证明产品已经完美。相反，它帮助发现：内容价值成立，但“从内容到成果”的路径还不够显性。

## 第四阶段：从课程货架到项目训练系统

产品方向从“增加更多课程”调整为：

```text
路径选择
  → 我的学习台
  → 重点课程
  → 操作清单
  → 课后任务
  → 提问与答疑
  → 用户成果
  → 下一步
```

北极星指标也从播放量和课程数量，转向：

> 每周完成一次“学习—动手—留下成果”闭环的活跃会员数。

## 第五阶段：知识站的正确位置

知识站不是业务起点。它是在以下条件出现后逐渐必要的：

- 已有稳定内容；
- 已有真实付费用户；
- 课程、任务和答疑重复出现；
- 用户需要持续更新；
- 需要统一管理会员、进度、材料和成果。

因此，给其他创作者的建议不是“复制一套知识站”，而是：

```text
先用内容确认问题
→ 用服务或小班完成收费验证
→ 用真实交付形成课程和任务
→ 用用户成果形成案例
→ 重复出现后再进入知识站
```

## 关键转折

### 转折 1：课程数量不是护城河

真正的护城河是持续判断、可执行解释、陪跑机制和用户成果。

### 转折 2：群不是完整产品

群适合即时互动和信号流，课程、更新、问题和成果仍需要进入稳定系统。

### 转折 3：工具权益不是赠品

当工具和额度可以降低真实试错成本时，它们是交付的一部分；但必须有明确使用场景。

### 转折 4：高评分不等于没有结构问题

正向反馈可以与路径混乱、视频体验和成果闭环不足同时存在。

### 转折 5：系统化不等于减少人的价值

系统承接重复动作，创作者本人继续提供判断、更新和高价值反馈。

## 可复用原则

1. 用户买的是结果和方向，不是内容数量。
2. 高接触交付是早期产品研究，不只是劳动。
3. 用户成果同时是交付证据、产品改进和增长资产。
4. 会员必须有持续问题和固定服务节奏。
5. 知识站只放大已经验证的交付闭环。
6. 创作者的判断要逐步变成路径、模板、评审标准和更新机制。

## 不可直接复制的部分

- 麦当的个人品牌与受众；
- 具体课程内容；
- AI 网关和麦子供应链；
- 飞书、微信和私域结构；
- 当前定价、优惠和运营节奏；
- 真实用户数据。

其他创作者应复制判断方法，不应复制实例配置。
